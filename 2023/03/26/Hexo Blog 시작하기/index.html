  • GitBlog 이용

시작하기전에 이미 Git Blog Repo를 만들어 놓아야 한다.

Git 설치하기

이때 설치되는 npm (Node.js Package Manager) 를 사용하여 hexo를 설치한다.

Blog 생성 및 연결

Blog를 생성할 폴더 위치를 정하고 이름을 정한다. 폴더 이름이 BlogRepo 이라면

  1. BlogRepo 폴더 , 해당 빈폴더 안에 들어가서 우클릭으로

    1. git bash 를 열어서
    2. hexo init 를 입력한다.

    그러면 hexo 에 필요한 기본 파일들이 생성된다. (git을 이용하여 생성됨)

  2. _config.yml 파일 수정

    맨 마지막 줄에 deploy 부분에 본인의 Git Blog 주소를 써준다. (이미 생성되어있어야함) 아래와 같이 수정해준다.

theme 적용하기

본인의 스타일에 맞는 theme 적용하기

theme마다 배포하는 방식이 조금 다르기 때문에 theme에서 제공하는 공식 메뉴얼을 따르는게 좋다(없어졌을수도있다)

Tip

  1. BlogRepo/_config.yml 에 themes: 적용할테마폴더이름 썻는지 확인해본다.
  2. theme를 바꾸고나서는 다음과 같이 clean 해주고 배포 해야한다.

Hexo BlogRepo 를 Git에 공유하기

먼저 themes를 git으로 다운받으면 themes 폴더에 .git이 남아서 정적인 상태로 git을 올릴수가 없다.
그러면 submodule이라는 상태로 공유되고 우리는 themes/_config.yml 의 파일 수정해야 하는데

그런 수정사항은 반영되지 않고 올라가게 된다.

따라서

themes폴더 적용된 테마 폴더안에 들어가 .git 폴더를 지운다

또한 올려진 BlogRepo를 다른곳에서 받았을 경우 다운받고 생성된 폴더에서 Git Bash를 열고 다음과 같이 hexo를 깔아야 한다
